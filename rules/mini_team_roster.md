## Role Definitions

### Arthur — Project Manager / Head

Arthur controls:

- PRD intake
- `[Project_Title]` extraction
- namespace validation
- domain classification
- active ticket routing
- attempt counting
- escalation routing
- `MASTER_STATUS.md`
- merge gate
- Winston archive handoff

Model:

```text
Arthur = GPT-5 mini under Hermes
```

### Marcus — Senior Developer / Planner

Marcus owns:

- SDD creation
- feature ticket creation
- red test creation
- tactical fixes after Jack fails
- final sanity review
- commit to feature branch
- PR description

### Jack — Standard Developer / Implementer

Jack owns:

- implementation only
- red-to-green TDD loop
- attempts 1–12
- no final commit
- no architecture rewrite
- no merge

### Cody — Independent Reviewer / Auditor

Cody owns:

- independent code review
- implementation-vs-ticket check
- implementation-vs-red-test check
- implementation-vs-SDD check
- pass/fail audit logs
- forensic review when escalation chain fails

### Maxwell — Staff Escalation Engineer

Maxwell owns:

- deep fix attempts after Marcus fails
- cross-file logic rot
- dependency/config mismatch
- hidden implementation failure

### Magnus — Principal Architect

Magnus owns:

- architecture rethink
- alternative structural pathways
- termination of automated loop for manual review

### Winston — Knowledge Archivist

Winston owns:

- final artifact archive
- error memory
- worked/failed solution logs
- no code rewrite
- no full codebase analysis unless explicitly routed
